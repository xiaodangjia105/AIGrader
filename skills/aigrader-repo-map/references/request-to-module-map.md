# 需求到模块映射

## 需求关键词 -> 主域

- "登录、token、当前用户、管理员、JWT" -> `aigrader-auth`
- "题库、题目、题型、批量导入、CSV" -> `aigrader-question-bank`
- "作业、创建作业、选题、组卷、截止时间" -> `aigrader-assignment`
- "提交、批改、评分、复核、订正、AI 评语" -> `aigrader-grading`
- "统计、班级统计、学生报告、准确率" -> `aigrader-statistics`
- "AI 配置、用户管理、质量监控、后台" -> `aigrader-admin`
- "接口、DTO、实体、一起改" -> `aigrader-change-playbook`
- "卡住、重复、评分不对、字段缺失" -> `aigrader-business-dictionary`

## 判定规则

- 只要出现题目、题型、选项、答案、难度，就先判定为题库域。
- 只要出现班级、截止时间、选题、组卷，就先判定为作业域。
- 只要出现提交、评分、反馈、复核、订正，就先判定为批改域。
- 只要出现登录、token、角色、权限，就先判定为鉴权域。
- 只要出现统计、报告、趋势、准确率，就先判定为统计域。

## 不要误判

- "看起来像作业列表" 也可能是批改列表；如果涉及评分和复核，主域仍是批改。
- "看起来像题目编辑" 也可能是作业内的题目；如果只是选题，主域仍是作业。
- "看起来像用户管理" 也可能是管理员操作；如果涉及权限变更，主域仍是 admin。
